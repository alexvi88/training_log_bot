# Анализ продукта и идеи улучшений

*Снимок на июль 2026. Цель — зафиксировать, что бот уже умеет, где осталось трение,
и собрать приоритизированный бэклог улучшений: отдельно «для удобства», отдельно
«для прикольности».*

---

## 1. Что продукт умеет сейчас (карта функциональности)

**Ядро — живое логирование тренировки** (`handlers/workout.py`, ~2.5K строк):

- Свободный текстовый ввод подходов: `100 8`, `100x8x3`, `+20 8`, `8` (свой вес),
  суффикс `@9` для RPE (`parser.py`). Вес можно опускать — возьмётся с прошлого подхода.
- Суперсеты: несколько «открытых» упражнений параллельно, переключение табами.
- Живой трекер пересоздаётся внизу чата после каждого подхода (`_refresh_live`),
  активное упражнение прижато к полю ввода.
- Подсказка «в прошлый раз» + подсказка прогрессии по двойной прогрессии 5–10
  повторов (`analytics.suggest_progression`), с галочкой «цель достигнута» прямо в ходе тренировки.
- Детект PR (вес, e1RM, повторы на весе, тоннаж) сразу на карточке завершения,
  сравнение роста e1RM с историческим максимумом, юбилейные тренировки
  (`analytics.is_workout_milestone`).
- Карточки упражнений с фото и описаниями (`exercise_descriptions.py` — 57K текстов,
  `media/exercises/` — сотни фото).
- Умные переходы: «следующее по шаблону», «как в прошлый раз» (`_suggested_next_exercise`).
- Завершение с заметкой, проверка «дата не сегодняшняя?», обработка зависших тренировок.

**Вокруг ядра:**

| Блок | Что есть |
|---|---|
| Программы | Свои рутины из прошлых тренировок + каталог готовых программ |
| История | Пагинация, просмотр, полное редактирование (сеты, дата), удаление, бэкфилл задним числом |
| Прогресс | Экран по упражнению (e1RM, рекорды, сессии), график с периодами 8/20/всё, теплокарта года со статами прямо в главном меню |
| Дневник веса | Логи, график по периодам, undo |
| Импорт/экспорт | CSV в обе стороны, импорт с интерактивным маппингом колонок |
| AI-тренер | Grok + 9 инструментов над данными юзера, голосовые вопросы (транскрибация), фото-вопросы, веб-поиск с дешёвым «стоит ли искать» гейтом, дневные лимиты, комментарий к тренировке (фоном, не блокирует завершение), AI-дайджест по воскресеньям |
| Пуши | Голос «ПРИВЕТ АТЛЕТ» (см. `PUSH_IDEAS.md`): серия на кону, пропуски, возвращение, тайминг, плато, аналитика, новичок. Ротация текстов без повторов, максимум 1 пуш/день, опт-аут |
| Шаринг | «Поделиться картинкой» — рендер карточки тренировки в PNG |
| Админка | Юзеры, их тренировки, диалоги с AI, лог пушей, дневной отчёт с учётом стоимости LLM-вызовов, бэкап БД |

**Инженерная база:** aiogram 3, SQLite, FSM переживает рестарты (JSON-стораж),
36 файлов тестов, CI, миграции «на лету» в `db.init_db`, аккуратный подход к
Telegram-ошибкам (`IgnoreStaleCallbackMiddleware`).

## 2. Сильные стороны (что не надо трогать)

1. **Скорость логирования — главная фича.** Текстовый ввод `100x8x3` быстрее любых
   кнопок-степперов из коммерческих приложений. Это ядро ценности.
2. **Единый голос продукта.** «ПРИВЕТ АТЛЕТ» в пушах, меню и онбординге — редкий
   случай, когда у пет-проекта есть характер. Это стоит усиливать (см. §5).
3. **Пуш-дисциплина.** 1 пуш/день, подколки только за пропуски, правило «пуш — только
   про то, чего не видно на экране» — грамотнее, чем у большинства фитнес-приложений.
4. **AI не для галочки:** инструменты над реальными данными юзера, гейт на дорогой
   поиск, учёт стоимости, фолбэки при отсутствии ключей.
5. **Обратная совместимость и живучесть:** стейт переживает рестарты, зависшие
   тренировки обрабатываются, стейл-коллбеки не роняют UX.

## 3. Где осталось трение (анализ по сценариям)

### Сценарий «я в зале, между подходами»

- **Нет таймера отдыха.** Самая частая механика силовой тренировки никак не
  поддержана: юзер не видит, сколько прошло с последнего подхода, и не может
  попросить напоминание «отдых кончился».
- **Повторить подход — надо печатать.** Самый частый кейс «тот же вес, те же повторы»
  требует ввода текста (`8` спасает, но кнопка была бы быстрее — руки в мелу).
- **Шаг веса зашит константой** (`_WEIGHT_STEP = 2.5 кг` в `handlers/workout.py`).
  Для гантелей шаг обычно 2 кг, для блоков — 5+, поэтому подсказка прогрессии
  иногда предлагает несуществующий вес.
- **Разминочные подходы не отличаются от рабочих** — они завышают тоннаж и могут
  ломать сравнение сессий (e1RM спасает `top_set`, но тоннаж и «в прошлый раз» — нет).

### Сценарий «начинаю тренировку»

- Если юзер не завёл программу, каждый раз путь: группа мышц → список → упражнение.
  Кнопки «Повторить прошлую тренировку» нет — а это, вероятно, самый частый паттерн
  реальных занятий (циклы A/B).
- Текстовый поиск упражнения работает только на экране списка (`pick_exercise_search`
  в состоянии `picking_exercise`) — на экране групп текст молча падает в fallback.

### Сценарий «смотрю на своё время»

- **Всё в серверном времени.** `dt.date.today()` и `ENGAGEMENT_HOUR` — локальные для
  сервера. Юзер из другого пояса получает пуш «обычно в это время ты под грифом»
  не в своё время, а «сегодняшняя» дата тренировки поздним вечером может уехать
  на завтра. Таблица `users` не хранит пояс.

### Мелкое

- Дата вводится только текстом `дд.мм.гггг` (быстрые кнопки — лишь сегодня/вчера/позавчера).
- Заметка бывает только у тренировки целиком — негде записать «болело плечо на жиме».
- В `config.py` остались мёртвые `FOLLOWUP_POLL_MINUTES` / `FOLLOWUP_DELAY_HOURS` —
  сама фича удалена (миграция в `db.py` дропает колонки).

## 4. Улучшения для удобства (приоритизировано)

### 4.1 Таймер отдыха — самое ценное недостающее

Минимальная версия дешёвая, потому что живой трекер и так пересоздаётся после
каждого подхода: у сетов есть время создания — достаточно дописать в хинт строку
«⏱ отдых 2:41» (время с последнего подхода) при каждом `_refresh_live`.
Вторая ступень — опциональное напоминание: после подхода планировать
`asyncio` задачу на N минут (настройка «⏱ Отдых: 2 мин» в карточке упражнения или
настройках), которая присылает «Отдых окончен — следующий подход» и сама
отменяется, если юзер уже залогировал новый сет. Никакой внешней инфраструктуры
не нужно.

### 4.2 Кнопка «🔁 Повторить подход»

В `logging_keyboard` рядом с «↩️ Удалить последний» — один тап логирует копию
последнего сета. В зале это заметно быстрее печати.

### 4.3 «Повторить прошлую тренировку»

Кнопка в главном меню (или на экране старта): стартует тренировку с планом из
последней завершённой — механика уже есть у рутин (`_reopen_exercises`,
«следующее по шаблону»), нужно только скормить ей прошлую тренировку как источник.
Закрывает главный сценарий юзеров без программ.

### 4.4 Часовой пояс пользователя

Колонка `users.tz_offset` (или именованный пояс), один вопрос при онбординге
(«Во сколько у тебя сейчас часов?» с кнопками) — и все `dt.date.today()` /
`ENGAGEMENT_HOUR` считаются в поясе юзера. Без этого пуш-тайминг и «дата
тренировки» врут всем не из пояса сервера.

### 4.5 Разминочные подходы

Префикс `w` / `р` в парсере (`w 60x8`) — сет пишется с флагом warmup: показывается
серым в трекере, исключается из тоннажа, PR и «в прошлый раз». Парсер расширяется
одной регуляркой, `sets` — одной колонкой.

### 4.6 Шаг веса на упражнение

Колонка `exercises.weight_step` (по умолчанию как сейчас, 2.5/5), правка из карточки
упражнения. Подсказка прогрессии станет точной для гантелей и блоков.

### 4.7 Заметка к упражнению в ходе тренировки

Кнопка «📝» на экране логирования — текст прицепляется к блоку упражнения и
показывается в «в прошлый раз» на следующей тренировке («болело плечо — следи
за локтями»). Это превращает дневник в настоящий журнал.

### 4.8 Календарь для дат

Инлайн-календарь (свой на InlineKeyboardBuilder, месяц 6×7) для бэкфилла и правки
даты вместо ручного `дд.мм.гггг`. Быстрые кнопки оставить.

### 4.9 Поиск упражнения текстом отовсюду

Разрешить текстовый ввод уже на экране групп мышц: искать по всем упражнениям
юзера + шаблонам, как сейчас в `pick_exercise_search`.

## 5. Идеи «для прикольности»

### 5.1 Реакция бота на PR — самый дешёвый дофамин

Telegram-боты умеют ставить реакции (`setMessageReaction`). Когда залогированный
подход бьёт рекорд — бот молча ставит 🔥 (или 🏆) прямо на сообщение юзера с
«120 5». Ноль лишних сообщений, мгновенная обратная связь, очень «живое» ощущение.
Дешевле фичи в этом списке нет.

### 5.2 Тоннаж-эквиваленты на карточке завершения

В пушах уже есть «за месяц поднял синего кита» (`engagement.format_tonnage`).
Перенести это на карточку завершения: «Сегодня: 6.4 т — это гружёная „Газель“ 🚚»
с ротацией сравнений (слон, рояль, вагон метро, МКС-модуль…) по той же механике
«сумки без повторов», что у пуш-текстов. Это то, чем делятся скриншотами.

### 5.3 Ачивки

Таблица `achievements` + экран «🏆 Достижения» в меню:

- **Клубы весов:** жим 100 / присед 140 / тяга 180 (по e1RM или чистому весу);
- **Стрик-медали:** 4 / 12 / 26 / 52 недели подряд;
- **Тоннаж за всё время:** 100 т «Синий кит», 1000 т «Товарный вагон»…;
- **Особые:** «Ранняя пташка» (тренировка до 7 утра), «Новогодний» (1 января),
  «Марафонец» (тренировка 2+ часа).

Выдача — при завершении тренировки, строкой на карточке (по правилу из
`PUSH_IDEAS.md`: юзер и так смотрит на экран — никаких пушей вдогонку).
Плюс шаринг-карточка ачивки — рендер уже есть (`charts.render_workout_card`),
нужен второй шаблон.

### 5.4 Годовой «Wrapped»

Раз в год (и/или по кнопке) — инфографика: суммарный тоннаж, топ-3 упражнения,
лучший стрик, самый сильный месяц, любимый день недели, все PR года. Теплокарта
уже рисуется (`charts.render_year_heatmap`) — это её звёздный час. Формат,
которым хвастаются в сторис, то есть бесплатный маркетинг.

### 5.5 Выбор персоны AI-тренера

Голос пушей — «строгий тренер из подвала», а комментарии AI — нейтральные.
Дать выбор в настройках: «Подвальный тренер» (дефолт, тот же голос, что в пушах),
«Спокойный методист», «Бро». Технически — один параграф в `_system_prompt()` +
колонка в `users`. Консистентный характер продукта — редкая и запоминающаяся вещь.

### 5.6 Рулетка тренировки

Кнопка «🎲 Мне повезёт» на старте: бот собирает случайную (но осмысленную —
1 группа мышц или пуш/пул из недавних) тренировку из каталога юзера. Для дней
«пришёл, но плана нет». Забавно + снижает порог старта.

### 5.7 Дуэли и челленджи (крупная ставка)

Deep-link приглашение другу → недельный челлендж «кто больше тоннажа / тренировок»
с финальной карточкой-сравнением. Социальная механика — главный недостающий
драйвер удержания, но требует аккуратности с приватностью (только взаимное
согласие, только агрегаты, без деталей тренировок).

### 5.8 Инлайн-режим для хвастовства

`@бот` в любом чате → отправить карточку последней тренировки или свежего PR.
Рендер карточек уже есть; инлайн-режим — это распространение продукта само по себе.

## 6. Технические заметки (попутно)

1. **Мёртвый конфиг:** `FOLLOWUP_POLL_MINUTES` / `FOLLOWUP_DELAY_HOURS` в `config.py`
   ссылаются на удалённую фичу — убрать.
2. **Монолиты:** `db.py` (64K) и `ai_trainer.py` (60K) просятся на разбиение по
   доменам (db/workouts, db/users, ai/tools, ai/search) — не срочно, но каждый
   новый домен (ачивки, челленджи) стоит начинать уже отдельным модулем.
3. **Часовой пояс** — см. §4.4: это и продуктовая, и техническая задача; чем позже
   делать, тем больше мест с `dt.date.today()` придётся трогать (сейчас их уже
   десятки по хендлерам).

## 7. Сводная приоритизация

| Горизонт | Что | Почему |
|---|---|---|
| **Быстрые победы** (часы) | 🔥 Реакция на PR (5.1) · 🔁 Повтор подхода (4.2) · Тоннаж-эквиваленты на финише (5.2) · Чистка мёртвого конфига (6.1) | Максимум ощущаемой пользы на строку кода |
| **Средние** (1–3 дня каждая) | ⏱ Таймер отдыха (4.1) · «Повторить прошлую тренировку» (4.3) · Разминочные сеты (4.5) · Шаг веса (4.6) · Персона AI (5.5) · Заметки к упражнению (4.7) · Календарь (4.8) · Поиск отовсюду (4.9) · Рулетка (5.6) | Закрывают главные точки трения ядра |
| **Крупные ставки** (недели) | Часовые пояса (4.4) · Ачивки (5.3) · Wrapped (5.4) · Инлайн-режим (5.8) · Дуэли (5.7) | Удержание, вирусность, корректность для всех регионов |

Рекомендуемый порядок старта: **5.1 → 4.2 → 5.2** (один короткий PR, весь про
«зал здесь и сейчас»), затем **4.1 таймер отдыха** как первая средняя фича — она
чаще всего упоминается как причина уйти в специализированные приложения.
