# Анализ продукта и идеи улучшений

*Версия 2, июль 2026. Разбор всей кодовой базы + прочёс фич конкурентов (Hevy, Strong,
Fitbod) и свежих возможностей Telegram Bot API. Цель — приоритизированный бэклог:
отдельно «для удобства», отдельно «для прикольности», отдельно «телеграм-нативное».*

---

## 1. Что продукт умеет сейчас (карта функциональности)

**Ядро — живое логирование тренировки** (`handlers/workout.py`, ~2.5K строк):

- Свободный текстовый ввод подходов: `100 8`, `100x8x3`, `+20 8`, `8` (свой вес),
  суффикс `@9` для RPE (`parser.py`). Вес можно опускать — возьмётся с прошлого подхода.
- Суперсеты: несколько «открытых» упражнений параллельно, переключение табами.
- Живой трекер пересоздаётся внизу чата после каждого подхода (`_refresh_live`),
  активное упражнение прижато к полю ввода.
- Подсказка «в прошлый раз» + подсказка прогрессии по двойной прогрессии 5–10
  повторов (`analytics.suggest_progression`), с галочкой «цель достигнута» в ходе тренировки.
- Детект PR (вес, e1RM, повторы на весе, тоннаж) сразу на карточке завершения,
  сравнение роста e1RM с историческим максимумом, юбилейные тренировки.
- Карточки упражнений с фото и описаниями (`exercise_descriptions.py` — 57K текста,
  `media/exercises/` — сотни фото).
- Умные переходы: «следующее по шаблону», «как в прошлый раз» (`_suggested_next_exercise`).
- Завершение с заметкой, проверка «дата не сегодняшняя?», обработка зависших тренировок.

**Вокруг ядра:**

| Блок | Что есть |
|---|---|
| Программы | Свои рутины из прошлых тренировок + каталог готовых программ (`seed_data.WORKOUT_PROGRAMS`) |
| История | Пагинация, просмотр, полное редактирование (сеты, дата), удаление, бэкфилл задним числом |
| Прогресс | Экран по упражнению (e1RM, рекорды, сессии), график с периодами 8/20/всё, теплокарта года со статами в главном меню |
| Дневник веса | Логи, график по периодам, undo |
| Импорт/экспорт | CSV в обе стороны, импорт с интерактивным маппингом колонок и резолвом названий упражнений |
| AI-тренер | Grok + 9 инструментов над данными юзера, голосовые вопросы (транскрибация), фото-вопросы, веб-поиск с дешёвым гейтом «стоит ли искать», дневные лимиты, комментарий к тренировке фоном, AI-дайджест по воскресеньям |
| Пуши | Голос «ПРИВЕТ АТЛЕТ» (`PUSH_IDEAS.md`): серия на кону, пропуски, возвращение, тайминг, плато, аналитика, новичок. Ротация без повторов, максимум 1 пуш/день, опт-аут |
| Шаринг | «Поделиться картинкой» — рендер карточки тренировки в PNG (`charts.render_workout_card`) |
| Админка | Юзеры, их тренировки, диалоги с AI, лог пушей, дневной отчёт с учётом стоимости LLM-вызовов, бэкап БД |

**Инженерная база:** aiogram 3, SQLite, FSM переживает рестарты (JSON-стораж),
36 файлов тестов, CI (ruff + pytest на 3.11/3.12), миграции «на лету» в `db.init_db`,
аккуратная работа со стейл-коллбеками Telegram.

## 2. Сильные стороны (что не надо трогать)

1. **Скорость логирования — главная фича.** Текстовый ввод `100x8x3` быстрее любых
   кнопок-степперов коммерческих приложений. Это ядро ценности.
2. **Единый голос продукта.** «ПРИВЕТ АТЛЕТ» в пушах, меню и онбординге — у продукта
   есть характер. Стоит усиливать (см. §5).
3. **Пуш-дисциплина.** 1 пуш/день, подколки только за пропуски, правило «пуш — только
   про то, чего не видно на экране».
4. **AI не для галочки:** инструменты над реальными данными юзера, гейт на дорогой
   поиск, учёт стоимости, фолбэки при отсутствии ключей.
5. **Живучесть:** стейт переживает рестарты, зависшие тренировки обрабатываются,
   стейл-коллбеки не роняют UX.

## 3. Где осталось трение (анализ по сценариям)

### «Я в зале, между подходами»

- **Повторить подход — надо печатать.** Самый частый кейс «тот же вес, те же повторы»
  требует ввода текста (`8` спасает, но одна кнопка быстрее — руки в мелу).
- **Шаг веса зашит константой** (`_WEIGHT_STEP = 2.5 кг` в `handlers/workout.py`).
  Для гантелей шаг обычно 2 кг, для блоков — 5+, поэтому подсказка прогрессии
  иногда предлагает несуществующий вес.
- **Разминочные подходы не отличаются от рабочих** — завышают тоннаж и портят
  «в прошлый раз» (e1RM спасает `top_set`, но остальное — нет).
- **Один сет — одно сообщение.** Нельзя вбить сразу «100 8, 100 7, 95 8» одной строкой.

### «Начинаю тренировку»

- Без заведённой программы каждый раз путь: группа мышц → список → упражнение.
  Кнопки «Повторить прошлую тренировку» нет, а циклы A/B — самый частый паттерн.
- Текстовый поиск упражнения работает только на экране списка (`pick_exercise_search`),
  на экране групп текст молча падает в fallback.

### «Смотрю на своё время»

- **Всё в серверном времени.** `dt.date.today()` и `ENGAGEMENT_HOUR` — локальные для
  сервера. Юзер из другого пояса получает пуш «обычно в это время ты под грифом»
  не в своё время, а вечерняя тренировка может уехать на завтрашнюю дату.
  Таблица `users` пояс не хранит.

### «Хочу понять картину целиком»

- Недельный объём по группам мышц считается (`analytics.classify_weekly_volume`,
  ориентиры 5–12 сетов/неделю) — но виден **только AI-тренеру** как инструмент
  `get_weekly_volume_by_group`. Юзер сам этот экран нигде не видит.
- Все рекорды разбросаны по карточкам упражнений — нет единого «зала славы».
- Тренд e1RM считается (`analytics.linear_trend`) — но никак не используется для
  прогноза, только для стрелочки.

### Мелкое

- Дата — только текстом `дд.мм.гггг` (быстрые кнопки лишь сегодня/вчера/позавчера).
- Заметка бывает только у тренировки целиком — негде записать «болело плечо на жиме»
  (при том, что колонка `exercises.notes` в схеме уже есть и не используется).
- Колонка `exercises.unilateral` тоже есть — но односторонние упражнения логируются
  как обычные, без «на руку/на ногу».
- В `config.py` мёртвые `FOLLOWUP_POLL_MINUTES` / `FOLLOWUP_DELAY_HOURS` — фича
  удалена (миграция в `db.py` дропает её колонки).

## 4. Улучшения для удобства (приоритизировано)

### 4.1 Кнопка «🔁 Повторить подход»

В `logging_keyboard` рядом с «↩️ Удалить последний» — один тап логирует копию
последнего сета. В зале это заметно быстрее печати.

### 4.2 Несколько сетов одной строкой

Разрешить в `parser.py` разделители запятая/перенос строки: `100 8, 100 7, 95 8` —
три сета одним сообщением. Дёшево (сплит перед `parse_single_token`) и сильно
ускоряет и живой лог, и бэкфилл.

### 4.3 «Повторить прошлую тренировку»

Кнопка в главном меню или на старте: тренировка с планом из последней завершённой.
Механика уже есть у рутин (`_reopen_exercises`, «следующее по шаблону») — нужно
скормить ей прошлую тренировку как источник. Закрывает главный сценарий юзеров
без программ.

### 4.4 Голосовой лог подхода

Транскрибация уже подключена для AI-тренера (`ai_trainer.transcribe_voice`).
Принимать войсы в состоянии логирования: «сто на восемь» / «жим сто на восемь три
подхода» → нормализация в токен парсера (можно дешёвой LLM-прослойкой с фолбэком).
В зале с мелом на руках голос удобнее клавиатуры. Это фича, которой нет ни у Hevy,
ни у Strong — телеграм-бот здесь может быть банально удобнее приложений.

### 4.5 Часовой пояс пользователя

Колонка `users.tz_offset`, один вопрос при онбординге («Сколько у тебя сейчас
времени?» с кнопками) — и все `dt.date.today()` / `ENGAGEMENT_HOUR` считаются в
поясе юзера. Чем позже делать, тем больше мест с `today()` придётся трогать.

### 4.6 Разминочные подходы

Префикс `w`/`р` в парсере (`w 60x8`): сет с флагом warmup — серым в трекере,
исключён из тоннажа, PR и «в прошлый раз». Одна регулярка + одна колонка в `sets`.

### 4.7 Экран «Недельный объём»

Данные и ориентиры уже есть (`_weekly_volume`, `classify_weekly_volume`, 5–12
сетов/группу) — осталось показать юзеру: экран в «Прогрессе» с полосками по группам
и метками «мало/в норме/много». У Hevy это платная фича («muscle heatmap»), здесь —
почти бесплатная, всё посчитано.

### 4.8 Шаг веса на упражнение

Колонка `exercises.weight_step` (дефолт как сейчас), правка из карточки упражнения.
Подсказка прогрессии станет точной для гантелей и блоков.

### 4.9 Калькулятор разминки и блинов

По целевому рабочему весу: «Разминка: 20×10 → 60×5 → 80×3 → 100» (процентная схема)
и раскладка блинов на гриф («по 25+15 на сторону»). Обе — любимые мелочи юзеров
Hevy/Strong, обе — чистая функция + строка в хинте логирования.

### 4.10 Заметка к упражнению в ходе тренировки

Кнопка «📝» на экране логирования — текст пишется в блок упражнения и показывается
в «в прошлый раз» на следующей сессии («болело плечо — следи за локтями»).
Колонка `exercises.notes` уже существует.

### 4.11 Быстрый лог без тренировки — «вбросить одной строкой»

Сообщение вида «жим 100х8х3, присед 140х5» вне активной тренировки → мгновенная
тренировка задним числом на сегодня. Резолв названий уже написан для CSV-импорта
(`handlers/exercise_resolve.py`) — тот же флоу, другой вход. Для тех, кто пишет
тренировку на бумажке, а вечером переносит.

### 4.12 Календарь для дат

Инлайн-календарь (месяц 6×7 на InlineKeyboardBuilder) для бэкфилла и правки даты
вместо ручного `дд.мм.гггг`. Быстрые кнопки оставить.

### 4.13 Поиск упражнения текстом отовсюду

Разрешить текстовый ввод уже на экране групп мышц — искать по всем упражнениям
юзера + шаблонам, как сейчас в `pick_exercise_search`.

### 4.14 Персональное расписание напоминаний

«Напоминай мне пн/ср/пт в 18:00» — личное расписание в настройках, отдельное от
умных пушей (и уважающее их правило «1 сообщение в день» — в день с напоминанием
умный пуш не шлём). Это самая запрашиваемая механика привычки в фитнес-приложениях.

### 4.15 Односторонние упражнения

Задействовать колонку `exercises.unilateral`: для них «в прошлый раз» и тоннаж
считать честно (×2 или лог по сторонам `100 8 л / 100 8 п` — решить по вкусу).

## 5. Идеи «для прикольности»

### 5.1 Реакция бота на PR — самый дешёвый дофамин

Боты умеют ставить реакции (`setMessageReaction`). Подход побил рекорд — бот молча
ставит 🔥 прямо на сообщение юзера «120 5». Ноль лишних сообщений, мгновенная
обратная связь. Дешевле фичи в списке нет.

### 5.2 Тоннаж-эквиваленты на карточке завершения

В пушах уже есть «за месяц поднял синего кита» (`engagement.format_tonnage`).
Перенести на карточку завершения: «Сегодня: 6.4 т — гружёная „Газель" 🚚» с ротацией
сравнений (слон, рояль, вагон метро, МКС-модуль…) по механике «сумки без повторов»
из `push_texts.py`. Этим делятся скриншотами.

### 5.3 Прогноз рекорда — «пророчество тренера»

`linear_trend` уже считает наклон e1RM. Если тренд стабильно положительный —
строка на экране прогресса: «При таком темпе 120 кг возьмёшь к марту». Почти
бесплатно (одна формула от текущего тренда до круглого числа) и очень мотивирует.
Голосом тренера, разумеется: «я такое видел — к марту пожмёшь».

### 5.4 «Зал славы» — экран всех рекордов

Один экран: лучший вес/e1RM по каждому упражнению, суммарный тоннаж за всё время,
лучший стрик, самая длинная тренировка. Плюс шаринг-карточка в стиле
`render_workout_card` (второй шаблон того же рендера).

### 5.5 Ачивки

Таблица `achievements` + экран «🏆 Достижения»:

- **Клубы весов:** жим 100 / присед 140 / тяга 180 (по чистому весу);
- **Стрик-медали:** 4 / 12 / 26 / 52 недели подряд;
- **Тоннаж за всё время:** 100 т «Синий кит», 500 т «Товарный состав», 1000 т…;
- **Особые:** «Ранняя пташка» (до 7 утра), «Новогодний» (1 января), «Марафонец»
  (2+ часа), «Коллекционер» (20 разных упражнений за месяц).

Выдача — строкой на карточке завершения (по правилу из `PUSH_IDEAS.md`: юзер уже
смотрит на экран, пуш вдогонку не нужен). Плюс шаринг-карточка ачивки.

### 5.6 Звания за стаж и тоннаж

Уровень как в старой качалке: «Новичок» → «Посетитель» → «Свой» → «Старожил» →
«Дед зала». Показывать в статах теплокарты главного меню. Тексты — тем же голосом,
что пуши; повышение звания — строка на карточке завершения.

### 5.7 Годовой «Wrapped»

Раз в год (и по кнопке) — инфографика: суммарный тоннаж, топ-3 упражнения, лучший
стрик, самый сильный месяц, любимый день недели, все PR года. Теплокарта уже
рисуется — это её звёздный час. Формат для сторис = бесплатный маркетинг.

### 5.8 Выбор персоны AI-тренера

Голос пушей — «строгий тренер из подвала», а комментарии AI — нейтральные. Дать
выбор в настройках: «Подвальный тренер» (дефолт), «Спокойный методист», «Бро».
Один параграф в `_system_prompt()` + колонка в `users`.

### 5.9 Рулетка тренировки

«🎲 Мне повезёт» на старте: случайная, но осмысленная тренировка (одна группа мышц
или пуш/пул с учётом того, что юзер давно не делал — `last_used_at` в `exercises`
уже хранится). Для дней «пришёл, но плана нет».

### 5.10 Фото прогресса + коллаж «до/после»

Отдельный раздел в дневнике веса: раз в неделю бот принимает фото (хранить только
`file_id`, файлы остаются у Telegram — БД не пухнет). По запросу — коллаж
«до/после» или сетка по месяцам через существующий matplotlib-рендер. Самая
эмоциональная фича трансформации — и её нет ни в одном телеграм-боте.

### 5.11 Замеры тела

Помимо веса: талия, бицепс, грудь, бедро (`body_measurements`, тот же флоу и график,
что у веса). У Hevy это в платной подписке; здесь — переиспользование готового
экрана дневника веса.

### 5.12 Дуэли и челленджи (крупная ставка)

Deep-link приглашение друга → недельный челлендж «кто больше тоннажа/тренировок»
с финальной карточкой-сравнением. Социальные стрики держатся на треть дольше
одиночных — это главный недостающий драйвер удержания. Только по взаимному
согласию, только агрегаты, без деталей тренировок.

### 5.13 Автоназвания тренировок

Вместо «14 марта» — «Грудь-пятница», «Ноги. Судный день», «Спина и немного гордости»:
генерится из групп мышц + день недели, ротация шаблонов как у пуш-текстов.
Появляется в истории, на карточках и в шаринге. Чистый вайб почти бесплатно.

## 6. Телеграм-нативные возможности (чего нет у приложений)

### 6.1 Инлайн-режим для хвастовства

`@бот` в любом чате → отправить карточку последней тренировки или свежего PR.
Рендер карточек готов; инлайн-режим — это распространение продукта само по себе.

### 6.2 Стриминг ответов AI-тренера

Bot API поддерживает стриминг/редактирование для AI-ответов — вместо «⏱️ отдыхаю
между подходами мысли, погоди...» печатать ответ по мере генерации. Ощущение
скорости — бесплатно.

### 6.3 План тренировки нативным чек-листом

Telegram ввёл чек-листы в сообщениях, и боты умеют их отправлять. Тренировка по
программе — как чек-лист: отметил упражнение — бот это видит и реагирует.
Экспериментально, но выглядит очень свежо для тех, у кого чек-листы доступны.

### 6.4 Мини-приложение для аналитики (крупная ставка)

Всё, что плохо живёт в чате — интерактивные графики, зум, сравнение упражнений,
календарь истории — идеально живёт в Telegram Mini App поверх тех же данных.
Чат остаётся быстрым логгером (это ядро!), Mini App — витриной прогресса.
Большая работа, но это естественный «второй этаж» продукта.

## 7. Технические заметки (попутно)

1. **Мёртвый конфиг:** `FOLLOWUP_POLL_MINUTES` / `FOLLOWUP_DELAY_HOURS` — убрать.
2. **Монолиты:** `db.py` (64K) и `ai_trainer.py` (60K) просятся на разбиение по
   доменам; каждый новый домен (ачивки, замеры, челленджи) стоит начинать отдельным
   модулем.
3. **Часовой пояс** — §4.5 одновременно продуктовая и техническая задача: мест с
   `dt.date.today()` по хендлерам уже десятки, дальше будет больше.

**Рекомендуемый старт:** пакет быстрых побед одним заходом — реакция на PR, повтор
подхода, несколько сетов строкой, тоннаж-эквиваленты и прогноз рекорда. Всё это
«зал здесь и сейчас» + эмоция на завершении, и всё опирается на уже существующие
расчёты. Следом — «Повторить прошлую тренировку» и голосовой лог: две фичи, после
которых бот становится удобнее мобильных приложений, а не «догоняет» их.
